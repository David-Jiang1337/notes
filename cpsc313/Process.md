In modern operating systems, a process is an abstraction used for CPU virtualization by providing programs the illusion that they have sole access to all of the resources of a computer. Processes provide the capability of running more than one program at a time on a computer, allow for easier development of programs as a given program does not need to consider the other programs running alongside it, and provides security via process isolation so that no individual program can take over the entire system without permission. The system of not permitting processes, by default, to know of or interact with each other whilst providing the illusion that no other processes exist on the same machine is known as process isolation.
## Address Space
Each process has its own address space, which is the range of valid locations in memory it can access. The address space of a process provides the illusion that the process has access to every possible memory address, without needing that capacity to be reflect in the hardware it is running on. To achieve this, the operating system provides the process with an address space via [[Memory Virtualization|memory virtualization]].
